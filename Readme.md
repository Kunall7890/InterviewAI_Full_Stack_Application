A comprehensive full-stack interview preparation application powered by Google Gemini AI. Prepare for technical interviews with AI-generated questions, real-time feedback, skill gap analysis, and personalized preparation plans.

## вњЁ Features

- **User Authentication**: Secure registration and login with JWT tokens and bcrypt password hashing

## рџљЂ Installation & Setup

### 1. Clone the Repository
```bash
git clone https://github.com/Kunall7890/InterviewAI_Full_Stack_Application.git
cd InterviewAI_Full_Stack_Application
```

### 2. Backend Setup

Navigate to the backend directory:
```bash
cd Backend
```

Install dependencies:
```bash
npm install
```

Create a `.env` file in the Backend directory with the following variables:
```env
PORT=5000
MONGODB_URI=mongodb://localhost:27017/interviewai
# OR use MongoDB Atlas:
# MONGODB_URI=mongodb+srv://<username>:<password>@cluster.mongodb.net/interviewai

GOOGLE_API_KEY=your_google_gemini_api_key_here
JWT_SECRET=your_secret_key_here
JWT_EXPIRE=7d
NODE_ENV=development
```

Start the backend server:
```bash
npm start
```

The backend will run on `http://localhost:5000`

### 3. Frontend Setup

In a new terminal, navigate to the frontend directory:
```bash
cd Frontend
```

Install dependencies:
```bash
npm install
```

Create a `.env` file in the Frontend directory:
```env
VITE_API_BASE_URL=http://localhost:5000/api
```

Start the development server:
```bash
npm run dev
```

The frontend will typically run on `http://localhost:5173`

## рџђ› Troubleshooting

### MongoDB Connection Issues
- Ensure MongoDB is running locally or Atlas connection string is correct
- Check network access settings if using MongoDB Atlas
- Verify `MONGODB_URI` in `.env` file

### API Key Issues
- Verify Google Gemini API key is valid and enabled
- Check API quota and rate limits
- Ensure API key has proper permissions

### CORS Issues
- Backend CORS should be configured to allow frontend domain
- Check backend `.env` configuration
- Verify frontend URL matches backend CORS settings

## How GenAI is used

- Backend service: \\`src/services/ai.service.js\\` uses \\`@google/genai\\`.
- \\`GoogleGenAI\\` client is initialized with \\`process.env.GOOGLE_GENAI_API_KEY\\`.
- Interview endpoint takes candidate input (resume, job details) and returns:
  - \\`matchScore\\`, \\`technicalQuestions\\`, \\`behavioralQuestions\\`, \\`skillGaps\\`, \\`preparationPlan\\`.
- Schema validated with \\`zod\\` in \\`interviewReport\\` model.

## Auth flow

- Register: hash password with \\`bcryptjs\\`, create user, issue JWT cookie.
- Login: validate credentials, issue JWT cookie.
- Protected routes: \\`auth.middleware\\` verifies token and checks blacklist.

## Troubleshooting
- \\`API key must be set\\` -> missing \\`GOOGLE_GENAI_API_KEY\\`
- \\`ECONNREFUSED _mongodb._tcp...\\` -> whitelist IP + cluster running
- \\`Invalid email or password\\` -> check DB records
- \\`JWT_SECRET\\` missing -> auth failing

- \\`npm run dev\\` (frontend)
